# Industrial AI OS Roadmap

路线图遵循“先闭环、再扩展”的原则。当前版本保留已有模块，不以增加菜单数量为目标，下一阶段重点提升数据可靠性、业务联动和真实部署能力。

## 当前基础

- CSV 订单导入基础版已完成
- 设备台账基础版已完成
- AI 生产计划闭环已完成
- Excel、PDF、OCR、PPT 核心演示流程已完成
- AI Gateway、Mock/API 模式和系统验收中心已建立

## Phase 1：CSV 导入强化

- 支持字段映射、编码识别、日期格式校验和导入错误报告
- 支持批量订单、库存、采购、发货模板
- 增加导入预览、撤销、重复订单检测和数据版本
- 将导入记录与操作日志、生产计划关联

验收目标：不同企业模板无需改代码即可完成字段映射和安全导入。

## Phase 2：设备台账强化

- 增加设备类型、产能、班次、可用工时、维护周期和停机原因
- 支持设备新增、删除、筛选、历史负载和维护记录
- 建立工艺与设备能力矩阵
- 排产时计算设备可用产能、换型时间和冲突

验收目标：生产计划能够解释每个工单为什么分配到特定设备。

## Phase 3：工单流转

- 订单转生产工单
- 工单拆分、合并、派工、开工、报工、完工和关闭
- 关联产品、BOM、工艺路线、设备、人员和质量记录
- 记录计划数量、完成数量、不良数量和实际工时
- 关键状态变更必须人工确认并写入审计日志

验收目标：从客户订单到生产完工形成可追踪的数据链路。

## Phase 4：报警中心

- 统一汇总延期、缺料、设备高负载、设备停机、质量异常和数据校验报警
- 支持级别、负责人、处理措施、截止时间和关闭状态
- Dashboard 展示实时报警指标和趋势
- 支持站内通知，预留邮件、企业微信、钉钉和飞书接口

验收目标：每条报警都有来源、责任人、状态和处理闭环。

## Phase 5：企业知识库

- 支持 PDF、Word、Excel、图片、SOP、工艺文件和设备说明书入库
- 扫描文档先 OCR，再执行切分、索引和检索
- 实现 RAG 问答、多文件问答和引用来源定位
- 增加文档权限、版本、标签、失效日期和重新索引

验收目标：回答具体业务问题时返回准确数据和可追溯来源，不凭空生成。

## Phase 6：真实 API 部署

- GitHub Pages 承载静态前端，Vercel 或 Linux 服务承载 API
- DeepSeek API Key 仅保存在服务端环境变量
- 完成 HTTPS、CORS、限流、日志、超时、重试和备用模型切换
- 将企业账号、订单、设备、工单和日志迁移到服务端数据库
- 增加自动化部署、健康检查、数据库迁移和备份恢复

验收目标：不同网络和设备通过同一公网网址登录，数据可同步，前端不包含任何密钥。

## 暂不优先

- 继续堆叠空菜单
- 无业务数据支撑的复杂大屏动画
- 未经人工确认自动修改关键生产数据
- 在公共演示环境上传真实涉密资料
